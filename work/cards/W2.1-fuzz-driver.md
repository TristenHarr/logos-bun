# W2.1 — P0.7 fuzz-driver + ddmin + regression-bank machinery (§8)

repo: logos-bun. The shared differential-fuzz driver EVERY P2+ component reuses.

## Deliverable (BAKE_A_BUN §8 is the spec)
- `conformance/fuzz-driver.mjs`: spawns two sides of a component's stdin→stdout protocol
  (the LOGOS probe `probes/<c>.lg` and the oracle shim `conformance/oracle/shims/<c>`),
  byte-compares, and on mismatch runs ddmin minimization, banking the minimized case in
  `fuzz/<c>/corpus/regressions/` FOREVER (replayed every gate run).
- ddmin (delta-debugging minimizer): generic, works on the byte/line/token input the
  component's PROBE.md declares.
- Corpus layout per §8: `fuzz/<c>/{PROBE.md, gen.mjs, corpus/{seed,regressions}}`.
- Replay: a `--replay` mode that runs every banked regression seed (deterministic) — this is
  gate check L13. Wire L13 into gate.sh (`l13()`, house style) — but GUARD it: no fuzz/<c>/
  dirs exist yet, so L13 passes trivially when the fuzz tree is empty (like l17's empty guard).
- Register the fuzz-tree guard in gate-manifest.json (coordinate with W1.6 — if W1.6 hasn't
  landed gate-manifest yet, note the entry to add; don't block).

## RED battery
`red/p0/fuzz/` — a TOY component (e.g. a trivial "reverse a string" probe + a matching-but-
deliberately-buggy oracle shim, OR two node stubs standing in for probe+shim since largo
probes need the toolchain): a seeded mismatch must (a) be detected, (b) ddmin-minimize to a
small witness, (c) bank in the regressions dir, (d) replay red on the next run, (e) once the
"bug" is fixed, replay green. Prove the full loop.

## Ratchet at exit
L13 (regression replay) in gate.sh; banked seeds are append-only (a gate-manifest guard).

## Manifest
conformance/fuzz-driver.mjs, fuzz/** (toy only for now), red/p0/fuzz/**, scripts/gate.sh (l13),
conformance/tests-shim-allowlist.tsv. Node driver is legit (npm-world tooling); toy probes may
be node stubs (real largo probes land at P2).
