## Deliverable
- `conformance/ledger/SCHEMA.md` — TSV schema:
  `STATUS ⇥ LANE ⇥ path[::name] ⇥ first-green-commit ⇥ asserts ⇥ note`
  Statuses: PASS / FAIL / BLOCKED(<gate>) / NOTIMPL / DIVERGE(reason) /
  QUARANTINE(expires=YYYY-MM-DD). Semantics per BAKE_A_BUN §6.3 verbatim
  (demotion path, promotion rules, incident requirements).
- `scripts/ratchet.mjs` — replays the PASS set; a PASS failure runs a confirmatory re-run
  on the same shard; confirmed → merge-freeze marker file + incident skeleton; unconfirmed
  → auto-demote to QUARANTINE(expires=+14d) + incident, repo stays open.
- `scripts/promote.mjs` — SOLE writer of PASS transitions; requires 5/5 passes recorded
  across ≥2 distinct run timestamps; writes first-green-commit + asserts count.
- `scripts/lints/ledger-lint.mjs` — validates schema, statuses, expiry dates, and the
  HASH CHAIN: every ledger file ends `#CHAIN <sha256(prev_chain ‖ body)>`; only
  ratchet/promote recompute it. Any hand edit breaks the chain = lint failure.
  PASS-set monotonicity vs `git show HEAD:<ledger>` (L2). `PASS →` transitions require
  an incident file + `ratchet-break` marker in the same commit.

## RED battery (commit these FIRST, watch each fail)
Fixtures in `red/p0/ledger/` driving the three scripts against toy ledgers:
1. confirmed PASS→FAIL turns ratchet.mjs exit nonzero + freeze marker
2. single unconfirmed flake auto-demotes to QUARANTINE + incident file written
3. hand-edited PASS row (chain broken) → ledger-lint fails
4. expired QUARANTINE → lint fails
5. PASS-count shrink vs HEAD → lint fails
6. promote.mjs refuses 4/5 or 5/5-in-one-run candidates
