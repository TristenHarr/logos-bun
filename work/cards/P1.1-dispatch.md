## ⚠️ TOOLCHAIN CONSTRAINTS at the pin (v0.10.1) — OBEY or the build won't parse (learned BUG-11):
1. `Repeat for x in xs:` loops parse ONLY inside `## To` functions, NOT at top level of `## Main`.
   → do all iteration in `## To` helpers.
2. A `.lg` module's abstract (paragraph after `# Title`) must contain markdown import-links OR
   be ABSENT. A link-less prose abstract breaks the parse. → title-only, or an abstract that is
   `Uses [Cli](file:./cli/dispatch.md).`-style links once you split into modules.
3. `args()` includes the binary path as item 1 (1-based, Rust convention). argv[1]="--version"
   is `item 2 of args()`. Iterate to be safe.
4. Equality is `x is equal to y` (verified) — `==` also works but prefer `is equal to`.
5. Multi-char identifiers (single-letter collection names have tripped the parser).

## Deliverable
`src/main.lg` + (as it grows) `src/cli/*.md` modules dispatching all 32 command tags from
vendor/bun/src/options_types/command_tag.rs. Map each subcommand's argv trigger string (read
vendor/bun/src/cli/Arguments.rs for the string→Tag mapping: `add`/`build`/`install`(+`i`)/
`run`/`test`/`x`(bunx)/`create`/`init`/`link`/`unlink`/`remove`(+`rm`)/`update`/`upgrade`/
`repl`/`exec`/`patch`/`outdated`/`publish`/`audit`/`why`/`pm`/`info`/… + argv0=="node" →
RunAsNodeCommand). Extract the real trigger strings — don't guess.

Behavior for the skeleton:
- `--version` / `-v` → the version (1.4.0), byte-exact vs the oracle.
- `--help` / `-h` / no-args → help text. Byte-exact --help is HARD (bun's is big); scope: emit
  a help string and exit 0, ledger the byte-exact match as a conformance row to grow toward
  (BLOCKED/partial is honest). Diff against the oracle to see the target.
- Each recognized subcommand → dispatch to a `## To` handler that emits the canonical
  `error: not implemented` and exits nonzero (P1.4 NOTIMPL — ledger a NOTIMPL row per command,
  so "missing" ≠ "broken" forever).
- Unknown flag → match bun's exit code (check the oracle: `bun --bogus`; echo $?).
