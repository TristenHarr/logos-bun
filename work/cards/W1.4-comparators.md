# W1.4 — P0.5 comparators: diffcli, normalize, treehash, exec-equivalence

repo: logos-bun

## Deliverable
- `conformance/oracle/diffcli.mjs` — run identical (argv, cwd, env) under oracle-bun and
  logos-bun; compare exit code + normalized stdout/stderr; emit a structured verdict.
- `conformance/normalize.ts` — port `normalizeBunSnapshot` from vendor/bun's harness
  (temp paths, PIDs, timings, version strings) + a PER-COMMAND normalizer allowlist file
  (`conformance/normalizers.tsv`, checked in; byte-exact wherever deterministic).
- `conformance/oracle/treehash.mjs` — canonical node_modules manifest: sorted walk of
  `relpath ⇥ mode ⇥ symlink-target|sha256`, emitted as a diffable manifest.
- exec-equivalence harness (bundler outputs may differ byte-wise; run the bundle under
  oracle-bun and node, byte-compare PROGRAM OUTPUT; decoded-sourcemap comparator stub).

## RED battery (FIRST)
- golden fixtures per normalizer class (a transcript containing temp paths/PIDs/timings →
  normalized golden).
- treehash on a fixture tree (incl. symlink + mode edge) → golden manifest; flip 1 byte in
  one file → manifest diff detected.
- diffcli self-test: oracle vs oracle == equal; oracle vs `sed`-tweaked wrapper == diff.

## Ratchet at exit
`conformance/normalizers.tsv` is append-visible under the ledger chain mechanism (W1.1):
adding a normalizer to hide a diff is a chained, visible event.
