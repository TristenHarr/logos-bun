# W1.5 — P0.10 workflow-ops harness: the runner discipline

repo: logos-bun

## Deliverable (scripts/workflow/)
- `commit.mjs` — THE canonical committer (CLAUDE.md R4): takes a card id + explicit path
  list; refuses paths outside the card's `## Manifest`; refuses `vendor/**` always;
  refuses implementation paths when the card's RED paths are not already committed
  (TDD at commit time, L10); runs `gate.sh --quick` before committing; only ever
  `git add <named> && git commit <named>`.
- `git-ro.sh` — read-only git verb allowlist (status/diff/log/show/rev-parse) for diff
  production and state reads.
- `diff.mjs` — emits `work/diffs/<task>-r<N>.patch` for reviewer consumption.
- `loop.mjs` — the §2.5 dynamic-workflow skeleton: reads a card, tracks
  QUEUED→RED→IMPL→REVIEW→FIX→GREEN in WAVES.md, stores loop artifacts under
  `work/loops/<task>/` (impl notes, review-1, review-2, fixer log).
- `cgroup-run.sh` — systemd-run wrapper (memory/CPU/pid caps) for stress-class tests;
  `offline-run.sh` — netns wrapper for hermetic tests.
- `scripts/lints/workflow-ops-lint.mjs` — greps scripts/** and work/cards/** prompts for
  forbidden verbs: `git stash|git reset|git checkout |git clean|git rebase|push --force|
  add -A|add \.|commit -a` (allowlist: this lint file itself + docs quoting them).

## RED battery (FIRST) — planted violations, each must be REJECTED
1. commit.mjs with an out-of-manifest path → refused
2. commit.mjs touching vendor/** → refused
3. commit.mjs with implementation file while card RED uncommitted → refused
4. a fixture script containing `git reset --hard` → ops-lint fails
5. loop.mjs state transition skipping REVIEW → refused

## Ratchet at exit
L8 (ops lint) + L10 (RED-first, enforced in commit.mjs) wired into gate.sh; plants 1–5
join the gate-audit (W1.6).
