# W2.8 — GIFT.3 gift review-gate wiring (§9.4 invariants 4-9, 14, 17, 18)

repo: logos-bun. Completes the gift pipeline: the pre-push gates that make a gift PR-ready.
Builds on W1.7 (upstream-gifts.tsv/ledger + templates already exist).

## Deliverable (BAKE_A_BUN §9.4 is the spec)
- `scripts/gift/preflight.mjs`: the pre-push checklist as a runnable gate for a candidate gift
  branch, mechanizing what's mechanizable:
  - invariant 4/5: a behavioral change ships an automated test in bun's format/folder
    (test/js/bun/, test/js/node/, test/cli/, test/bundler w/ itBundled); regression tests go in
    test/regression/issue/<REAL-N> ONLY for true regressions w/ a real issue number.
  - invariant 6: the test must FAIL under `USE_SYSTEM_BUN=1` and PASS under `bun bd test` — mark
    these two as `[USER]`-run steps (they need the gift checkout + a built bun; preflight EMITS
    the exact commands + a checklist the user runs, and records the result).
  - invariant 7: non-flaky (no setTimeout; N reruns) — a lint over the test source.
  - invariant 14: `bun run rust:check-all` cross-platform — `[USER]`-run, emitted.
  - invariant 17: the standing open-PR cap (default ≤ N, configurable) — preflight refuses to
    mark a new gift `ready` if N are already open (reads upstream-gifts.tsv state).
  - invariant 18: the §2.5 adversarial loop (2 diff-only reviewers + fixer) + comments ≤3 lines;
    regression tests get exactly one comment (the issue URL) — a lint over the prepared PR body.
  - invariant 13: license/CLA confirmation — a `[USER]`-confirmed gate (clean-room MIT, nothing
    from BSL logos-bun/logicaffeine sources).
- Triage classification (invariant 15) as the fuzz-driver's first post-mismatch step: a
  `classify` helper stamping ours/theirs/spec-ambiguity into upstream-gifts.tsv (via the W1.7
  gifts tooling — do NOT reimplement; call it).
- Gate check L18 (gift-preflight) OR fold into L17's gift lint — pick the cleaner wiring; if a
  new check, empty-guard it (no candidate gifts yet → passes trivially). GIFT.4 stays open.

## RED battery
`red/p0/gift-preflight/`: a candidate with a regression test named test/regression/issue/FAKE
→ rejected (needs a real N); a behavioral change with no test → rejected; an over-cap state
(N+1 gifts ready) → refuses the new ready; a flaky test (setTimeout) → rejected; a valid
candidate → passes (with the [USER] steps clearly emitted, not auto-passed).

## Ratchet at exit
L18 (or L17-extension) in gate.sh; the open-PR cap enforced; [USER] steps emitted not faked.
